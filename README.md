# AI Flaws Repository

Documentation of AI system limitations, flaws, and problematic behaviors encountered in production use.

## Purpose

This repository serves as a record of AI system behaviors that represent limitations, design flaws, or violations of user rights and expectations. Each documented case includes:

- Date and context of the incident
- Description of the AI behavior
- User's perspective on why this constitutes a flaw
- Impact on workflow and user autonomy

## Documented Cases

- [API Key Processing Refusal](./incidents/api-key-refusal-2026-07-01.md) - Oz AI agent refuses to process user's own API key, citing security concerns

## Contributing

Each incident should be documented with full context, including conversation logs where possible, to provide a complete record of the AI system's behavior.
